Database Concurrency · Task 6

Lost Update Simulator

Two transactions read the same price, update it independently, and one overwrites the other. Watch the race condition unfold in real-time — then enable Safe Mode to see locking prevent it.

🔵
Transaction A — Slow
Reads the price, sleeps 2 seconds, then updates to its value. By the time it wakes up, Transaction B has already committed.
🔴
Transaction B — Fast
Reads the same initial price, immediately updates and commits. Its change is completely overwritten by A — a Lost Update.
🔒
Safe Mode — FOR UPDATE
A acquires a row-level lock. B blocks and waits. Sequential execution — no data loss, no conflict.
